# Intentionally orphan snippets — allowlist for scripts/check-snippet-wiring.py
#
# Each row whitelists a snippet that exists in snippets/*.liquid but isn't
# rendered/included anywhere in the codebase. The lint exits non-zero on
# any orphan NOT in this list, catching the yxte (2026-05-18) regression
# class where a refactor silently dropped a {% render %} callsite.
#
# Format (whitespace-separated; the reason continues to end-of-line):
#   <snippet-path>   <bd-id>   <target_wire_by_YYYY-MM-DD>   <reason>
#
# target_wire_by is a soft signal: rows past their date emit a STALE warning
# but the lint still passes. The intent is to surface stale dormant snippets
# for triage (delete or wire), not to block merges.
#
# Categories (informal — used to keep this file scannable):
#   [TAE]      = moved to a Theme App Extension; snippet is dormant by design
#                (the app loads itself via the TAE injection point now)
#   [P4-PORT]  = P4 Pipeline 8 migration in progress; snippet awaits a port
#                or is intentionally deferred per the migration plan
#   [DEAD]     = candidate for deletion; was used in Pipeline 6 but P8
#                migration replaced the functionality
#   [DYNAMIC]  = rendered via {% render <variable> %} or content_for_*; the
#                lint's regex (literal strings only) doesn't catch the
#                reference
#   [INVESTIGATE] = unclear why orphan; needs a bd ticket to investigate
